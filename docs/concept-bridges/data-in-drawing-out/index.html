Data In, Drawing Out

A concept bridge for data-driven drawing. Real numbers — temperatures, counts, scores — become visual shapes when you map their values to visual properties.

Data is for spreadsheets. I'm not sure how numbers connect to visual sketches.
After this bridge
A number can control size, color, height, or opacity. Mapping data values to visual properties turns any dataset into a picture.

Big idea Data visualization is choosing which visual property carries meaning. Size communicates magnitude. Color communicates intensity. There is no single correct encoding.
⚠ Common mistake Using color alone to encode numbers excludes colorblind viewers. Always pair color with another channel — size, position, or a label.
Try this Set data to 0. Which encoding completely disappears? Which stays visible even at zero?

Numbers become shapes when you decide what they mean

Data-driven drawing is the choice to let a number control a visual property. The data doesn't change; the drawing code doesn't know where the numbers come from. You provide the mapping. A temperature becomes a bar height. A score becomes a circle size. A percentage becomes opacity. Once you see this translation clearly, any dataset becomes drawable.

Teacher move

Prompt Go to Compare and change one value in the data. Which shape changes, and in what way?
Misconception Students think data "becomes" the drawing. Clarify: data is mapped to visual properties — height, width, color — through a deliberate translation. The data doesn't draw itself; the sketch decides what to do with each number.
Ask What would you encode as height? As color? What would that picture tell you about the data that a table of numbers wouldn't?
