Variable Scope: Where Variables Live

A concept bridge for understanding where to declare variables in p5.js. Where you create a variable controls where it can be used and how long it remembers its value.

Big idea Variables declared with let inside draw() are created fresh every frame and destroyed when that frame ends. They have no memory.
⚠ Common mistake The most common beginner mistake: declaring a counter or position variable inside draw(). It always resets — nothing moves, nothing accumulates.

Declare variables where they need to live

Every variable has a scope — the region of code where it exists. Variables declared with let inside a function only exist during that function call. Variables declared inside a for loop only exist during the loop. Variables declared outside all functions (global scope) persist for the entire sketch.

The most important rule for p5.js: if a value needs to survive from one frame to the next — position, score, color, state — declare it globally.

Teacher move

Prompt Ask learners to write a prediction: if let count = 0 is declared inside draw() and count++ follows it, what will count show on frame 1000? Collect the predictions, then run it and ask someone to explain what actually happened.
Misconception Students see "x is not defined" and assume they misspelled the variable. Often the real cause is scope: the variable was declared inside one function and read inside another. Each function has its own local scope — a variable born in setup() is invisible in draw().
Ask You want a ball to cross the screen — x increases by 2 each frame. Where should you declare x: inside draw(), inside setup(), or outside both? What breaks in each of the wrong two cases, and why?
