World vs Local Coordinates

A concept bridge for translate(), rotate(), push(), and pop(). In p5.js, every shape lives in a coordinate system — and you can move that system before drawing.

Move the paper, then draw simply

World coordinates are the canvas's default system. translate() moves the origin to a new spot, so you can draw at (0,0) and have it appear anywhere. rotate() spins the local grid around that new origin. push() and pop() keep these transforms from affecting the rest of your sketch. Together, they let you build and move complex objects with simple local math.

Teacher move

Prompt Go to translate() and move the origin to a corner. Now add rotate(). What is the shape rotating around?
Misconception Students think translate() "moves the shape." Clarify: it moves the origin — the point where (0, 0) lives. The shape is always drawn at the same local coordinates; you're repositioning the floor it stands on.
Ask What would happen if you called rotate() before translate() instead of after? Try to predict the result, then run it.
