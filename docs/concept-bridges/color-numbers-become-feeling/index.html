Color: Numbers Become Feeling

A concept bridge for fill(), stroke(), grayscale, RGB, HSB, alpha, and lerpColor(). In p5.js, color is not magic — it is numbers interpreted through a color model.

Before this bridge, you might think
Color in code is arbitrary numbers you have to memorize. There is no way to feel your way through it.
After this bridge
Color is a small set of numbers — brightness, light channels, hue, or transparency — and each model gives you a different kind of expressive control.

Grayscale uses one number

Think of grayscale as a brightness dial. Turn it toward 255 and the shape glows white; turn it toward 0 and it disappears into darkness. This is the simplest color mode — one number, one dimension, full control.

Teacher move

Prompt Set all three HSB sliders to their midpoint. Drag only the Hue slider from one end to the other. What changes and what stays the same?
Misconception Students often confuse Saturation and Brightness. Clarify: Saturation is vividness (gray → pure color), Brightness is lightness (black → full value). You can have a dark vivid color or a light washed-out one — they're independent axes.
Ask If you want two colors that are "the same vibe but different brightness," which slider stays fixed and which do you move? Now try it.
