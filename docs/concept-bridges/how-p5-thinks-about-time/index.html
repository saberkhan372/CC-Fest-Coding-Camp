How p5.js Thinks About Time

A concept bridge for setup(), draw(), frameCount, frameRate(), and trails. Animation is not continuous magic — it is repeated drawing.

setup once → draw again and again
30 fps
255
Frame rate presets:
setup() runs once. draw() runs over and over. That repetition creates animation.

setup() happens once. draw() repeats.

p5.js starts with setup(), then enters the draw loop. Every time draw() runs, p5.js makes another frame.

setup() → draw() → draw() → draw() → ...

Big idea

Animation is repeated drawing.

frameCount is how many drawings have happened.

frameRate() controls how many drawings p5.js tries to make per second.

Legend
