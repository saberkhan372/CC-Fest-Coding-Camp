How p5.js Thinks About Time

A concept bridge for setup(), draw(), frameCount, frameRate(), and trails. Animation is not continuous magic. It is repeated drawing, one frame at a time.

Before this bridge, you might think
Animation is one smooth thing that “just happens,” and draw() is only where motion code belongs.
After this bridge
A sketch is a loop of frames. setup() prepares once, draw() repeats, and each frame redraws the current state.

A sketch is a repeated conversation with the screen

p5.js does not move shapes for you. It redraws the sketch over and over. If values change a little each frame, motion appears. If the background is not fully cleared, old frames linger as trails.
