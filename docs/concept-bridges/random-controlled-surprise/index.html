random(): Controlled Surprise

A concept bridge for random(). You choose the range — the computer chooses one value from inside it. Every call gives a different result.

Live coderandom(min, max) returns a decimal in range.
Big idea random(min, max) returns a decimal number ≥ min and < max. The range is half-open: min is possible, max is never returned.
⚠ Common mistake random() with no arguments returns 0.0–1.0. random(255) returns 0–255. random(255, 0) with max < min silently reverses the range in p5.js.
Try this Set min and max to the same value. What happens? Now make the range very small (like 100 to 101). How often do you get the same value twice?

You choose the range; the computer chooses the value

random() is a tool for designed surprise. You provide the boundaries — a range, a set of choices, a grid size — and random() picks something inside them. The result is unpredictable but controlled.

Call it once in setup() for permanent randomness. Call it in draw() for continuous variation. Use floor() to convert decimals into integer indices. Use randomSeed() when you want the same surprise every time.
