map() Range Translator

A concept bridge for map(). One number from an input range becomes a matching number in an output range. This is how mouse position, time, sound, and data start controlling size, color, height, transparency, and motion.

data scaling mouse control visual translation animation mapping interactive color
Before this bridge, you might think
map() is a weird formula you memorize, or a magic trick that somehow changes numbers into visuals.
After this bridge
map() is a translator. It keeps the same relative position while moving a value from one range into another.
Fuzzy idea map() feels like a formula to memorize. See it Watch one range translate into another range. Change it Adjust input, output minimum, and output maximum. Code idea map() keeps the same relative position in a new range. Go next Remix size, color, data bars, opacity, and motion controls.

The same relative position moves to a new world

If an input is 25% of the way through its original range, map() places the output 25% of the way through the new range. That is the whole idea. Once you feel that relationship, map() stops being a formula and starts becoming a wiring move.

Teacher move

Prompt Set the output range from 0 to 20. Move the input slider all the way across. Now change the output max to 200. What's the same between the two runs, and what changed?
Misconception Students expect map() to clamp. It doesn't — if the input goes past its stated range, the output goes past too. Use constrain() afterward, or toggle "Constrain output" here and predict what happens before you do.
Ask If the input is exactly halfway through its range and you swap the output min and max, where does the output land? Write your prediction, then try it.
