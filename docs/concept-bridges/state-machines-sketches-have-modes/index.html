Which mode is your sketch in right now?

Every interactive sketch has modes — sometimes called states, phases, or screens. A state machine makes those modes explicit by storing the current mode in a variable and checking it once at the top of draw(). The rest of the code only runs when the sketch is in the right state. Adding a new mode means adding a branch, not a tangle of booleans.

Teacher move

Prompt The sketch has three modes: title, playing, game over. Click through each one. What single variable decides which mode you're in — and where does it change?
Misconception Students think they need a separate sketch for each screen. Clarify: one state variable and a few if/else checks handle everything. The canvas doesn't know about screens — only your code does.
Ask What happens if you forget to change state after a collision? The game keeps playing. Draw the transition diagram first — missing arrows become the bugs.
