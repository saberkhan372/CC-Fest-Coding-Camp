Speed
Notice this: random() fills evenly — every position is equally likely, so the pile stays flat. randomGaussian() piles up in the center, forming a bell. noise(t) drifts: consecutive values are similar, so the pile shifts across the canvas in slow runs instead of filling uniformly.
Break it on purpose: Set speed to ×2 and watch all three reach 1000 dots. The random pile stays roughly flat. The Gaussian bell becomes clear early and changes very little after 300 dots. The noise pile never fully settles — reset and watch it drift a different direction each time.
Remix this: In p5.js, try let x = randomGaussian(width/2, 60) to place shapes that cluster near the canvas center. Great for portrait sketches, organic layouts, and anything that shouldn't look evenly scattered.

Teacher move

Prompt Before running: ask students to predict which pile will be tallest in the center after 500 dots. Write it down, then check. Then ask: what shape is each pile?
Misconception noise() is just smoothed random() — the output is still uniformly spread. It isn't: consecutive noise values are correlated, so dots fall in local runs instead of independent positions.
Ask After 1000 dots: which distribution would you use to make a circle appear near the canvas center? Which for a field that fills every corner? Which for a walking agent?
