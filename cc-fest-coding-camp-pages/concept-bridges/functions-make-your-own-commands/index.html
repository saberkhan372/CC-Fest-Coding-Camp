Repeated code is a clue

When the same group of drawing commands appears again and again, you probably have an idea that wants to become a function.

circle(200, 200, 40); line(200, 220, 200, 270); circle(320, 200, 40); line(320, 220, 320, 270);

Big idea

Function: a custom command.

Parameter: a placeholder inside the function.

Argument: the real value passed into the function call.

Return: sends a calculated value back out.

Functions are most useful when they name an idea. Good function names often sound like commands: drawFlower, makeButton, checkCollision, calculateScore.
