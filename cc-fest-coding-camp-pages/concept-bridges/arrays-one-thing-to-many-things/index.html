Arrays: One Thing to Many Things

A concept bridge for using arrays to map data to visuals. One value drawn once. Same structure drawn many times. The loop reads each item; the drawing code scales with it.

Before this bridge, you might think
I draw one circle with one set of numbers. To draw five circles I copy the code five times.
After this bridge
Store five values in an array. One loop draws one shape per value. Change the array and the picture changes.

The same drawing code, different data

A bar chart and a field of circles are the same structure: an array of values, a loop, one drawing command. The values change what you see. The code stays the same. This is why data-driven drawing is powerful — you design one shape, then the data multiplies it. Want ten shapes? Add ten values. The loop handles the rest.

Teacher move

Prompt The sketch draws circles from an array. Add a 6th value at the end. Which circle is new — and how do you know? Now change the first value. Which circle moved?
Misconception Students think the index controls position on screen. Clarify: the index is just order in the list. The value at that index drives the visual — you decide what to do with each value inside the loop.
Ask If you wanted all 44 circles to grow 1px every second, would you write 44 lines of code or loop through the array once? What does each approach cost when you need to change something?
