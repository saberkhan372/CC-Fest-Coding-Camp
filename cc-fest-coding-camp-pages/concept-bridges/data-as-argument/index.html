Data as Argument

A concept bridge for passing data into functions as arguments. The function decides how to draw; the caller decides what data to use — this separation makes functions reusable.

The function decides how; the caller decides what

When a function accepts data as arguments, it separates two concerns: how to draw stays inside the function; what data to use comes from the caller. This makes functions truly reusable. Pass a different array and you get a different chart. Pass a different object and you get a different creature. The drawing code never needs to know where the data came from.
