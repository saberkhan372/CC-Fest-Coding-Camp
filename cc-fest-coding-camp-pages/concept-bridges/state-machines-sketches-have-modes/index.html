A concept bridge for state variables, start/play/win/lose screens, and complete mini-games. A state machine helps a sketch know what mode it is in and what should happen next.

Current state:
A state variable stores the current mode. The sketch draws and behaves differently depending on that state.

One variable controls the mode

A state variable stores what mode the sketch is currently in. Common game states are start, play, win, and lose.

let gameState = "start";

Big idea

State: the current mode of the sketch.

Screen: what gets drawn for a state.

Transition: a rule that changes one state into another.

State machine: the map of possible states and transitions.
