Events: Sketches Listen

A concept bridge for mousePressed(), keyPressed(), mouseMoved(), and mouseReleased(). Events let your sketch respond to moments, not just the passage of time.

⚠ Common mistake
Try this

Events fire once; draw() runs forever

In p5.js, draw() runs 60 times per second no matter what. Event functions run only when something happens. mousePressed() fires once when the mouse button goes down — not 60 times per second, just once. Use events to record moments. Use draw() to display them.

Teacher move

Prompt Hold the mouse button and drag slowly across the canvas for 3 seconds. Predict how many times mousePressed() fires before you release. Then check the click counter and compare.
Misconception Students use mouseIsPressed inside draw() expecting it to behave like mousePressed() — but mouseIsPressed is true every frame the button is held, not once per click. One captures a continuous state; the other captures a single moment.
Ask You want a light toggle: click once to turn on, click again to turn off. Would you put the flip inside mousePressed() or check mouseIsPressed inside draw()? What single line of code inside mousePressed() would switch the state?
