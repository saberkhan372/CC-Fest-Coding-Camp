One pattern, any number of things

An array stores a collection. A for loop visits every item. Together, they turn one circle into a field of 50, one particle into a system, one data point into a bar chart. The power is not in the loop itself — it is that the same code works whether your array has 2 items or 2000. Adding one more item is just one push(), and the loop handles the rest.

Teacher move

Prompt Click Push several times and watch the array grow. Now drag the sliders — every item updates at once. What's doing that work?
Misconception Students often think the loop knows which items changed and redraws only those. Clarify: the loop visits every item, every frame. Five items or five hundred, the same code runs — the cost is linear, not selective.
Ask What would happen if you removed the for loop and called draw on just circles[0]? Would the other circles appear on screen?
